# 验证清单

## 1. 用户验收清单

- `FR-01` 用户能看到三套预设日程模板：工作日、普通休息日、节假日。
- `FR-02` 系统能先判断当天日类型，再选择对应模板。
- `FR-03` 节假日能覆盖周末模板，单天覆盖又能覆盖默认判定。
- `FR-04` 系统能根据模板生成当天提醒计划，而不是要求用户逐条重新配置。
- `FR-05` 到点后系统会实际发送提醒，而不只是页面里存在配置。
- `FR-06` 用户不回复时，系统仍会在等待窗口后给出最低动作。
- `FR-07` 用户能通过一键按钮反馈，不需要输入长文本。
- `FR-08` 系统能根据状态输出正常版、降级版或保底版动作。
- `FR-09` AI 下一步安排不会输出长规划，不会要求开启新项目。
- `FR-10` 用户点击“我已经滑坡了”后，系统会执行恢复协议，而不是指责用户。
- `FR-11` 用户可以看到当日执行记录。
- `FR-12` 晚上能收到收口建议和明日第一步。

## 2. 关键路径测试

### Case 1：工作日模板被正确选中

步骤：

1. 准备一个普通周三日期，不在节假日表里。
2. 配置工作日、周末、节假日三套模板。
3. 执行 `prepare_day_schedule(date)`。

预期：

- 返回 `dayType = workday`
- 选中工作日模板
- 生成的提醒计划来自工作日模板

### Case 2：周末模板被正确选中

步骤：

1. 准备一个普通周六日期，不在节假日表里。
2. 执行 `prepare_day_schedule(date)`。

预期：

- 返回 `dayType = weekend`
- 选中周末模板
- 生成的提醒计划数量和时间符合周末模板

### Case 3：节假日覆盖周末

步骤：

1. 准备一个周六日期。
2. 同时把这一天加入节假日表。
3. 执行 `resolve_day_type(date)`。

预期：

- 返回 `holiday`
- 不使用周末模板

### Case 4：单天覆盖优先级最高

步骤：

1. 准备一个普通周三日期。
2. 为该日期配置单天覆盖为 `holiday`。
3. 执行 `prepare_day_schedule(date)`。

预期：

- 返回 `dayType = holiday`
- 选中节假日模板

### Case 5：模板缺失回退工作日模板

步骤：

1. 准备一个周末日期。
2. 删除或停用周末模板。
3. 执行 `prepare_day_schedule(date)`。

预期：

- 系统不崩溃
- 回退到工作日模板
- 记录模板缺失告警或回退信息

### Case 6：15:00 正常提醒链跑通

步骤：

1. 预置一个包含 15:00 提醒项的工作日模板。
2. 物化当天计划。
3. 运行调度器到 15:00。
4. 检查是否创建 `ReminderRun`。
5. 检查是否发送提醒消息。

预期：

- 运行记录状态正确
- 推送消息含可访问链接

### Case 7：无回复自动推进

步骤：

1. 触发一次提醒。
2. 在等待窗口内不提交任何反馈。
3. 执行无回复处理任务。

预期：

- 系统生成 `minimum` 级别动作
- `ReminderRun` 被标记为自动推进

### Case 8：点击“继续安排”

步骤：

1. 打开提醒反馈页。
2. 点击“继续安排”。
3. 检查返回的 `ActionPlan`。

预期：

- 输出时长在 20 到 60 分钟之间
- 输出包含明确动作和停止条件
- 输出不包含“开启新项目”

### Case 9：点击“我已经滑坡了”

步骤：

1. 打开提醒反馈页。
2. 点击“我已经滑坡了”。

预期：

- 系统返回恢复协议
- 动作中包含离开当前位置和写下第一步
- 计划时长不超过 20 分钟

## 3. 边界条件测试

- 工作日、周末、节假日三类日期都能至少找到一套模板。
- 同一分钟内多个提醒项到期时，系统应分别生成运行记录。
- 同一提醒运行重复提交反馈时，系统应保留历史并只将最后一个计划视为当前计划。
- 用户备注为空时，系统仍应能生成计划。
- 健康状态为空时，系统应退回通用动作。
- 晚间收口在当天没有任何反馈时，仍应能输出最小收口建议。
- 同一日期重复物化计划时，不应产生重复提醒项。

## 4. 失败模式测试

- 推送渠道不可用时，系统要记录失败而不是静默丢失。
- LLM 超时时，系统要回退规则模板。
- 数据库写入失败时，接口要返回错误并记录日志。
- 提醒链接 token 无效或过期时，页面要给出明确错误提示。
- 同一提醒运行被重复执行自动推进时，不应重复生成多条相同保底动作。
- 节假日日期表缺失时，系统要回退工作日模板或明确告警。
- 模板被全部停用时，系统要明确报错而不是生成空白计划。

## 5. 手工验证步骤

### 每日自用验证

1. 上午确认今天被判定为哪种日类型。
2. 检查今天实际加载的是哪套模板。
3. 下午 15:00 等待提醒是否送达。
4. 至少测试一次“无回复”路径。
5. 至少测试一次“继续安排”或“想逃避”路径。
6. 晚上确认是否收到收口建议。
7. 第二天回看是否能看到昨天的执行记录。

### 连续 7 天观察指标

- 模板判定是否符合预期
- 15:00 后直接滑坡次数
- 15:40 后成功启动 20 分钟任务的次数
- 晚上报复性工作次数
- 晚上刺激娱乐时长是否下降
- 身体不适时是否更快切换为恢复动作
- 用户主观感受是否是“被帮助”，而不是“被轰炸”
