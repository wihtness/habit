# 核心流程伪代码

## 1. 解析当天日类型

```text
function resolve_day_type(date, holiday_dates, day_overrides):
    if date in day_overrides:
        return day_overrides[date]

    if date in holiday_dates:
        return "holiday"

    if date.is_weekend():
        return "weekend"

    return "workday"
```

输入：

- 当前日期
- 本地节假日日期表
- 用户单天覆盖配置

输出：

- `workday | weekend | holiday`

关键规则：

- 单天覆盖优先级最高
- 节假日优先级高于普通周末
- 默认工作日回退

## 2. 选择日程模板

```text
function select_schedule_template(day_type, templates):
    selected = templates.find_enabled_by_day_type(day_type)

    if selected exists:
        return selected

    fallback = templates.find_enabled_by_day_type("workday")

    if fallback exists:
        return fallback with warning("template_missing_fallback_to_workday")

    raise TemplateSelectionError("no_enabled_template")
```

输入：

- 当天 `day_type`
- 可用模板列表

输出：

- 选中的 `ScheduleTemplate`

异常路径：

- 对应模板缺失时，优先回退工作日模板
- 工作日模板也缺失时，明确报错，不能静默失败

## 3. 物化当天提醒计划

```text
function materialize_day_schedule(date, template):
    reminder_plans = []

    for item in template.items where item.is_enabled:
        reminder_plans.add(
            ReminderPlan(
                templateItemId = item.id,
                planDate = date,
                name = item.name,
                time = item.time,
                type = item.type,
                goalContext = item.goalContext,
                defaultActionLevel = item.defaultActionLevel,
                feedbackWindowMinutes = item.feedbackWindowMinutes,
                allowLlm = item.allowLlm
            )
        )

    return reminder_plans sorted by time, sortOrder
```

输入：

- 当前日期
- 选中的模板

输出：

- 当天 `ReminderPlan[]`

关键规则：

- 只物化启用中的提醒项
- 保留模板项顺序
- 允许不同模板拥有不同数量和不同时间的提醒项

## 4. 生成当天调度结果

```text
function prepare_day_schedule(date):
    dayType = resolve_day_type(
        date = date,
        holidayDates = holidayRepo.listDates(),
        dayOverrides = overrideRepo.listOverrides()
    )

    template = select_schedule_template(
        dayType = dayType,
        templates = templateRepo.listAll()
    )

    reminderPlans = materialize_day_schedule(date, template)

    reminderPlanRepo.replace_for_date(
        date = date,
        reminderPlans = reminderPlans
    )

    return PreparedDaySchedule(
        date = date,
        dayType = dayType,
        templateId = template.id,
        reminderPlans = reminderPlans
    )
```

输出：

- 当天日类型
- 选中的模板
- 当天全部提醒计划

关键规则：

- 每天开始时或系统首次运行时执行
- 同一天重复生成时应幂等，避免重复计划

## 5. 到点触发提醒

```text
function run_due_reminders(now):
    duePlans = reminderPlanRepo.find_due(
        date = now.date,
        time = now.time
    )

    for plan in duePlans:
        run = reminderRunRepo.create(
            planId = plan.id,
            scheduledAt = combine(plan.planDate, plan.time),
            triggeredAt = now,
            status = "pending_send",
            feedbackDeadline = now + plan.feedbackWindowMinutes
        )

        message = notificationComposer.build(plan, run.token)
        result = notifier.send(message)

        if result.success:
            reminderRunRepo.markSent(run.id, result.messageId)
        else:
            reminderRunRepo.markSendFailed(run.id, result.error)
            notifier.retryOnce(message)
```

## 6. 无回复自动推进

```text
function auto_progress_no_reply(runId, now):
    run = reminderRunRepo.findById(runId)

    if run.status not in ["sent", "waiting_feedback"]:
        return

    if feedbackRepo.existsForRun(runId):
        return

    context = buildRuleContext(
        run = run,
        feedbackType = "no_reply"
    )

    fallbackPlan = ruleEngine.generate(
        context = context,
        intensity = "minimum"
    )

    actionPlanRepo.create(
        runId = runId,
        source = "rule",
        intensityLevel = fallbackPlan.level,
        planText = fallbackPlan.text,
        durationMinutes = fallbackPlan.duration,
        stopCondition = fallbackPlan.stopCondition
    )

    reminderRunRepo.markAutoProgressed(runId)
    notifier.send(fallbackPlan.followupMessage)
```

## 7. 用户反馈处理

```text
function submit_feedback(runId, feedbackType, note, bodySignals):
    run = reminderRunRepo.findById(runId)

    feedback = feedbackRepo.create(
        runId = runId,
        feedbackType = feedbackType,
        note = note,
        bodySignals = bodySignals
    )

    context = buildPlanningContext(run, feedback, bodySignals)

    if feedbackType == "slipped":
        plan = ruleEngine.buildSlipRecovery(context)
    else:
        rulePlan = ruleEngine.generate(context)

        if shouldCallLlm(context, rulePlan):
            llmPlan = llmPlanner.generateNextStep(context, rulePlan)
            plan = normalize_or_fallback(llmPlan, rulePlan)
        else:
            plan = rulePlan

    actionPlanRepo.create(plan)
    reminderRunRepo.markResponded(runId)

    return plan
```

## 8. LLM 输出规范化

```text
function normalize_or_fallback(llmOutput, rulePlan):
    if llmOutput is invalid:
        return rulePlan

    if llmOutput.durationMinutes < 10 or llmOutput.durationMinutes > 60:
        return rulePlan

    if llmOutput.startsNewProject == true:
        return rulePlan

    if llmOutput.stopCondition is empty:
        return rulePlan

    return llmOutput
```
